What it does

Four things. One wall with a gate.

Every capability ships in the first commit. No promises, no roadmap vapour.

ALLOWED Allowlist

Name the tools, domains, and endpoints each agent may touch. Everything else is denied by default. That's the whole posture.

DENIED Redaction

Secrets and personal data are stripped from outbound calls before they leave your network.

HELD Risk gate

Destructive actions — deletes, spend, sends — don't just run. They stop and wait for a human to approve or reject.

ALLOWED Signed audit log

Every call, allowed or denied, becomes a signed, immutable record — exportable as compliance evidence, not just a log file.

Quickstart

Point your agent at the gateway. Change one line.

Docker
docker run -p 8080:8080 \
  -v ./policy.yaml:/policy.yaml:ro \
  ghcr.io/vystartasv/bastion-gateway
policy.yaml
default: deny
agents:
  researcher:
    allow:
      - GET api.internal/*
    hold:
      - POST api.internal/orders/*
      - DELETE *

Set your agent's base_url to http://localhost:8080. Done. Full docs on GitHub →

Where it sits

Agent infrastructure has layers.

Most tools live at the top — orchestration, memory, skills. Bastion Gateway lives lower: who this agent is (L2) and is it allowed to do this (L7). That's the gap between an agent that works and an agent you can put in production.

The Agent OSI model names the layers. Bastion Gateway fills two of them.

Open source
